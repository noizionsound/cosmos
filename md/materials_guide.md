# COSMOS — Инструкция по подготовке материалов

## Структура папки

```
ECHO/cosmos/sources/
├── audio/        ← амбисоник аудио (WAV)
└── photo/        ← текстуры и изображения
```

---

## Аудио

### Формат
- **WAV**, stereo или B-Format амбисоник (первый порядок: W, X, Y, Z)
- Sample rate: **48kHz** (или 44.1kHz — оба работают)
- Bit depth: 24-bit
- Длина: любая — движок зациклит

### Именование файлов

| Назначение | Имя файла | Описание |
|---|---|---|
| Фоновый амбиент мира | `world_amb.wav` | Общий фон всего мира, тихий, зацикленный |
| Амбиент пузыря I | `bubble_1_amb.wav` | Слышен при приближении к порталу I |
| Амбиент пузыря II | `bubble_2_amb.wav` | То же для II |
| Амбиент пузыря III | `bubble_3_amb.wav` | То же для III |
| Внутри пузыря I | `world_1.wav` | Главная музыка уровня I |
| Внутри пузыря II | `world_2.wav` | То же для II |
| Внутри пузыря III | `world_3.wav` | То же для III |
| Дополнительные текстуры | `texture_01.wav` … | Случайные 3D клики, слои |

### Как работает пространство
Движок использует **Web Audio API HRTF** — слушается в наушниках как настоящее 3D. Файлы из `sources/audio/` будут загружены как `AudioBuffer` и воспроизводиться через `PannerNode` с позицией в пространстве.

Если у тебя есть **B-Format амбисоник** (4-канальный: W/X/Y/Z) — скажи, подключим через специальный декодер. Если просто stereo — тоже хорошо, пойдут через стандартный HRTF паннер.

## Текстуры / Фото

Все изображения кладёшь в `sources/photo/`.

### Текущие файлы
| Файл | Используется | Назначение |
|---|---|---|
| `stone_texture.jpg` | ✅ да | Фон главного мира (тайлуется) |
| `man.jpg` | ❌ нет | Персонаж теперь рисуется в коде |

### Что можно добавить

| Назначение | Имя файла | Формат |
|---|---|---|
| Фон мира внутри пузыря I | `world_1_bg.jpg` | JPG или PNG, любой размер |
| Фон мира II | `world_2_bg.jpg` | то же |
| Фон мира III | `world_3_bg.jpg` | то же |
| Дополнительная текстура земли | `ground_2.jpg` | для вариации фона |
| Объекты / пропсы | `prop_stone.png` | PNG с прозрачностью |

### Рекомендации по текстурам
- **Тёмные и фактурные** — движок накладывает затемнение поверх, слишком светлое будет выглядеть бледно
- **Повторяющийся паттерн** — текстуры тайлуются, поэтому края должны стыковаться
- **Размер не критичен** — 500–2000px на сторону, больше не нужно (движок сам скейлит)

## Тексты (нарратив)

Тексты из `ECHO/text-corpus/` можно вставлять в игру. Когда персонаж входит в пузырь — текст появляется на экране.

Подготовь для каждого мира короткий фрагмент (2–5 предложений). Формат — обычный `.txt` или `.md`, положи в `sources/text/`:

```
sources/text/
├── world_1_text.md     ← текст для пузыря I
├── world_2_text.md     ← текст для пузыря II
└── world_3_text.md     ← текст для пузыря III
```

Можно брать прямо из уже написанных файлов vault — например из `text-corpus/author-text/Земля пьёт воду.md`.
